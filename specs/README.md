# COD-AB Specification

This specification defines the format, schema, and data standards for the Common Operational Dataset – Administrative Boundaries (COD-AB). It is the authoritative reference for what a valid COD-AB dataset must contain and how its data must be structured.

It does not describe operational workflows, software procedures, or publishing steps — those are covered in the COD-AB Geodatabase Development Guidance documents.

## Requirement Language

The key words **MUST**, **MUST NOT**, **REQUIRED**, **SHALL**, **SHALL NOT**, **SHOULD**, **SHOULD NOT**, **RECOMMENDED**, **MAY**, and **OPTIONAL** in this specification are to be interpreted as described in [RFC 2119](https://datatracker.ietf.org/doc/html/rfc2119):

- **MUST / MUST NOT / REQUIRED**: Absolute requirements. A dataset that violates these is non-compliant.
- **SHOULD / SHOULD NOT / RECOMMENDED**: Strong recommendations. Departures are permitted but require documented justification.
- **MAY / OPTIONAL**: Permitted but not required.
